## 2. 措施

### 2.1 AI的代码加注释（类注释 或者 方法注释 或者 代码块注释）

注释信息包括：代码生成时的时间戳、代码用途或者是为了解决什么问题或者是为了修复什么bug 等等

### 2.2 记录AI的提交树

记录AI提交的代码所在文件路径、git哈希值 等等信息

### 2.3 AI提交模版

AI提交的message以“feat(AI):” 或 “fix(AI):” 开头，后面接提交信息；并添加提交时的时间戳。
而程序员提交的message不作要求

### 2.4 给AI建独立的git仓库成员

当前每个程序员的电脑上的git用户名和邮箱都设置的是全局的即global，所以AI每次提交都要带上AI自己的git用户名和git邮箱

### 2.5 把提交代码的skill添加到AI工作流

工作流调用写代码的skill(phaseA)生成代码完成之后，再调用 [测试 | 编译 | 打包] 的skill（如果有的话）之后，立即调用git提交代码的skill(phaseN)，尽可能地缩短phaseA到phaseN的时间，降低程序员手工添加或修改代码的可能性。
